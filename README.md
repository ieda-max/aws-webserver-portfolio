# AWS Lightsailを用いたWebサーバー構築（HTTPS対応）

## ■ 概要

AWS Lightsail上にLinuxサーバーを構築し、ドメイン設定からWebサーバーの公開、HTTPS化まで一連の構築を行いました。
また、構築過程で発生したトラブルに対して原因を切り分け、解決まで対応しました。

---

## ■ 構成図

```
[ユーザー（ブラウザ）]
        ↓ https://iemn.link
[Route53（DNS）]
        ↓
[Lightsail（サーバー）]
        ↓
[Nginx（Webサーバー）]
        ↓
[HTML]
```

---

## ■ 使用技術

* AWS（Lightsail）
* Route53（DNS）
* Nginx（Webサーバー）
* Let's Encrypt（SSL証明書）
* Linux（Ubuntu系）

---

## ■ 実装内容

1. LightsailでLinuxサーバーを作成
2. Route53でドメインを取得し、AレコードでIPアドレスと紐付け
3. Nginxをインストールし、Webサーバーを構築
4. Certbotを使用してHTTPS化を実施
5. ブラウザから https://iemn.link でアクセス確認

---

## ■ 工夫した点

* 問題発生時にいきなり修正するのではなく、原因の切り分けを行ってから対応
* エラーメッセージやログをもとに原因を特定
* 一つずつ検証しながら確実に解決

---

## ■ 発生したトラブルと対応

### ① ポート競合エラー

* 内容：Nginx起動時にポート80が既に使用されており起動失敗
* 原因：Dockerコンテナがポートを使用していた
* 対応：Dockerコンテナを停止・削除し解決

---

### ② Docker権限エラー

* 内容：dockerコマンド実行時にpermission denied
* 原因：ユーザーにDocker実行権限がなかった
* 対応：sudoを使用して実行し解決

---

### ③ Nginx起動失敗

* 内容：nginx.service failed
* 原因：設定途中の不整合
* 対応：設定のリセットおよび再起動で復旧
